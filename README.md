## 🔧 How observability-toolkit Works

observability-toolkit collects data about your systems and software. It tracks metrics like CPU usage, memory, and response times. It uses Prometheus exporters to gather this data, which you can then view with monitoring tools like Grafana.

It also includes chaos engineering tests. These tests introduce small failures or slowdowns in your system to see if your alerts can detect problems early. This way, you catch issues before they affect users.

## 📚 Additional Information

### What Is Prometheus?

Prometheus is an open-source system that collects and stores metrics. Think of it as a way for your computer to keep track of how well it is working.

### What Is Chaos Engineering?

Chaos engineering involves intentionally causing small problems in a system to test how well it copes. This helps find weak spots before they become real issues.

### Why Use observability-toolkit?

- To monitor your system's health easily  
- To test your system’s alerting with chaos engineering  
- To get useful data presented clearly  
- To ensure your system reacts properly to failures
